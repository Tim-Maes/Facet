# Security Policy

## Supported versions

Facet is actively maintained on the current major release line and the default development branch.

| Version | Supported |
| ------- | --------- |
| 6.x     | Yes       |
| < 6.0   | No        |

## Reporting a vulnerability

Please do **not** report security vulnerabilities through public GitHub issues, discussions, or Discord.

Use one of these private channels instead:

1. GitHub's private vulnerability reporting for this repository, if it is enabled.
2. Direct private contact with the project lead at https://github.com/Tim-Maes.

When reporting an issue, include:

- A description of the vulnerability and the affected package or component.
- The versions, target frameworks, and environment involved.
- Reproduction steps or a proof of concept, if available.
- Any suggested mitigations or workarounds you have identified.

We will aim to acknowledge new reports within 5 business days and will coordinate remediation and disclosure with the reporter when a vulnerability is confirmed.

## Disclosure process

Security fixes may be developed privately until a patch or mitigation is ready. Once a fix is available, the project will publish the update and share any relevant upgrade guidance.
